## 1. Core Thesis

The ADI Pod has 200K+ words of timestamped, attributed, practitioner-grade AI engineering discussion sitting in markdown files. This is an extraordinary asset that is currently invisible to search engines and AI assistants. Most competing podcasts have short show notes. ADI Pod can have the most comprehensive, linkable, citable practitioner content on dozens of high-intent developer topics.

The strategy: build a static site that programmatically generates pages from transcript data, use AI agents to produce all content (your job is review, not writing), and establish ADI Pod as the definitive source for terms and concepts the show has coined or covered first.

**Why this works for a solo operator:** The existing agent skill infrastructure (podcast-transcripts, write-like-shimin, linkedin-marketing, podcast-hook-finder) becomes the content factory. The upfront investment is building the site and a content generation skill. After that, the weekly cycle is ~60-90 minutes of reviewing agent output plus ~75 minutes of distribution.

## 4. Agent Content Pipeline

### Existing Skills Leveraged

| Skill | Role in Pipeline |
|-------|-----------------|
| `podcast-transcripts` | Search, summarize, cross-reference transcript content |
| `write-like-shimin` | Voice consistency on all public-facing content |
| `podcast-hook-finder` | Short-form video moments with timestamps (ep 13-20 done) |
| `linkedin-marketing` | LinkedIn post generation in 7 formats (ep 12-17 done) |
| `podcast-critique` | Quality feedback loop for future episodes |
| `cheatsheet-generator` | Technique reference content |
| `influence-advisor` | Persuasion analysis for content framing |

### New Skill: `site-content-generator`

Wraps the pipeline into single commands:

```bash
# Generate all content for one episode
./scripts/generate.sh ep-20

# Batch generate glossary entries from all episodes
./scripts/generate-glossary.sh

# Generate/update a pillar page
./scripts/generate-pillar.sh claude-code-guide

# Generate a blog post
./scripts/generate-blog.sh "ai-code-quality-bugs"

# Generate segment hub content
./scripts/generate-segment.sh two-minutes-to-midnight

# Generate YouTube metadata (title, description, tags, chapters)
./scripts/generate-youtube-meta.sh ep-20
```

Each script:
1. Uses `podcast-transcripts` search to pull relevant transcript sections
2. Loads `write-like-shimin` voice profile for tone consistency
3. Generates markdown with correct Astro frontmatter
4. Outputs to `drafts/` directory for review
5. Approved drafts moved to `content/`, auto-deploys via git push

### Per-Episode Content Cycle

When a new episode drops, run the pipeline and review:

| Content Piece | Generator | Review Time |
|--------------|-----------|-------------|
| Show notes page (2,000 words, SEO-optimized) | site-content-generator | 10 min |
| 1-2 blog post drafts (deepest topics) | site-content-generator | 15 min each |
| New glossary entries (any coined terms) | site-content-generator | 2 min each |
| FAQ additions to relevant pillar pages | site-content-generator | 5 min |
| Pillar page updates (new insights) | site-content-generator | 10 min |
| Two Minutes to Midnight tracker update | site-content-generator | 5 min |
| 3-5 LinkedIn posts (7 format system) | linkedin-marketing | 10 min |
| Short-form video hooks with timestamps | podcast-hook-finder | 5 min |
| YouTube metadata (title, desc, tags, chapters) | site-content-generator | 5 min |

**Total per episode: ~60-90 minutes of review.**

### Backfill Strategy

For existing 20 episodes, run batch generation:
- 20 show notes pages — review in batches of 5
- 25+ glossary entries — generate all at once, quick scan
- 6 pillar pages — one deeper review each
- 6 segment hubs — review per segment
- 15+ blog posts — generate over 2-3 weeks, review 2-3 per session

Backfill is the bulk of upfront work but it's review, not writing.
